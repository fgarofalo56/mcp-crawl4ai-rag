## Initialization Sequence

### Phase 1: Project Structure
I'll analyze the project structure and identify key directories.

### Phase 2: Documentation Review
I'll read CLAUDE.md, README.md, and CONTRIBUTING.md to understand:
- Project purpose and goals
- Development setup requirements
- Team coding standards
- Testing requirements
- Deployment procedures

### Phase 3: Technology Assessment
I'll examine configuration files to identify:
- Programming languages and frameworks
- Dependencies and their versions
- Build and deployment tools
- Testing frameworks
- CI/CD pipelines

### Phase 4: Codebase Analysis
Using Serena, I'll search for:
- Entry points and core modules
- API endpoints and routes
- Database schemas and models
- Shared utilities and helpers
- Test coverage

### Phase 5: Current Work Status
I'll check `/project_tracking` for:
- Active tasks and assignments
- Blocked items
- Open bugs and issues
- Upcoming features
- Technical debt items

### Phase 6: Team Insights
I'll provide:
- Project health assessment
- Areas needing attention
- Testing gaps
- Documentation needs
- Suggested improvements

## Team Best Practices

### Context Management
- Use `/clear` between feature branches
- Use `/compact` during long sessions
- Monitor with `/context` regularly

### Code Quality
- Follow team style guide (see CONTRIBUTING.md)
- Maintain or improve test coverage
- Update documentation with changes
- Run linters before committing

### Communication
- Update CLAUDE.md with architectural decisions
- Document complex logic in code comments
- Keep project tracking updated
- Share context with team members

### Git Workflow
- Create feature branches from main/develop
- Write clear commit messages
- Request reviews before merging
- Keep branches up to date

---

## Error Recovery

If any step fails:
1. Check if files exist in expected locations
2. Verify Serena is configured and accessible
3. Ensure MCP servers are running
4. Review project structure for non-standard setup
5. Ask team lead for clarification
